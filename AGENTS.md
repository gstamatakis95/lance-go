# AGENTS.md: working on lance-go

Canonical guide for coding agents (and humans) working in this repo.

## What this is

Go bindings for the Lance columnar format: a Go package (`lance/`) over a
Rust FFI shim (`rust/`) that statically links the `lance` crate (pinned
`=8.0.0`). Data crosses via the Arrow C Data Interface. Params cross as
JSON. Errors cross as thread-local code+message.

## Build & test

Prereqs: Go 1.25, Rust >= 1.94 (CI pins 1.94.0), `protoc` on PATH.

```sh
make rust              # build the shim (regenerates include/lance_go.h).
                       # Cold: ~6-25 min; shim-only rebuilds: seconds.
make go-test           # go test ./... -race -count=1
make go-test-cgocheck  # GOEXPERIMENT=cgocheck2 go test ./... -count=1
make rust-test         # Rust workspace unit tests in the release profile
make test              # rust build + Rust tests + both Go test runs
make lint              # cargo fmt --check, cargo clippy -D warnings, go vet
make header-check      # verify the checked-in header matches cbindgen output
make platform-info     # print consumer CGO_CFLAGS/CGO_LDFLAGS
make docs              # regenerate docs/api.md from the package godoc
make docs-check        # verify docs/api.md is current (CI gates this)
make artifacts         # fetch prebuilt release artifacts (consumers)
```

Object-store tests (S3/Azure/GCS emulators, gated off by default):

```sh
make object-store-up                              # SeaweedFS + Azurite + fake-gcs
LANCE_GO_OBJECT_STORE_TESTS=1 go test ./... -run ObjectStore
make object-store-down
```

(`make test-object-store` wraps the middle step.)

## Hard rules

1. **Never edit `include/lance_go.h` by hand.** It is cbindgen output,
   regenerated by `rust/build.rs` on every `make rust`. Change the Rust
   source, rebuild, and commit the regenerated header together with the
   Rust change. CI fails on drift (`git diff --exit-code include/lance_go.h`).
2. **Never run a bare `cargo update`.** `rust/Cargo.lock` pins
   `time = 0.3.47` to work around a rustc-1.94 / aws-smithy E0119 trait
   conflict. Update individual crates with `cargo update -p <crate>` and
   keep the `time` pin intact.
3. **All Arrow data exported to the native side must be C-allocated.**
   Public callers should use `lance.Allocator()`; in tests, always use
   `internal/testutil.Allocator()`, never `memory.DefaultAllocator`.
   Violations crash under `GOEXPERIMENT=cgocheck2`, which CI runs.
   (In Go 1.25 `GOEXPERIMENT=cgocheck2` replaces `GODEBUG=cgocheck=2`.)
4. **Uphold the FFI conventions** (see `rust/src/lib.rs`):
   `lance_` prefix, i32 error codes + thread-local message, owned strings
   freed via `lance_string_free`, JSON for option structs, native side
   takes ownership of exported Arrow structs even on error.
5. **Go errors wrap exactly one sentinel** from `lance/errors.go`, with the
   `"lance: <verb>:"` prefix added exactly once at the wrap layer, so
   `errors.Is` classification keeps working and strings never double-prefix.
   All fallible FFI calls MUST go through `ffiCall(ctx, ...)` (which pins
   the OS thread across the call and the thread-local error retrieval, and
   installs the ctx cancellation token). Never call a fallible `C.lance_*`
   function directly. New/migrated terminal methods should be thin bodies
   passed to the `datasetOp`/`datasetDo`/`fragmentOp` helpers in
   `lance/ops.go` (see the contract in that file's doc comment); stream-
   returning ops and result-metric ops stay hand-rolled by design.

6. **Every exported symbol carries a godoc comment** that starts with the
   symbol name and opens with a one-sentence summary. Document sentinel
   errors for common failure cases, Close/Release obligations on returned
   handles/readers, and deviations from the package-wide ctx-cancellation
   contract — but do not restate package-wide defaults per method. When the
   public surface changes, run `make docs` and commit the regenerated
   `docs/api.md` (CI fails on drift via `make docs-check`).

7. **Callback-bridge rules** (`lance/callbacks.go` + `rust/src/callbacks.rs`,
   the Go-plugin FFI layer, see the module docs for details):
   - Payload ownership: Go allocates output buffers with `C.malloc`
     (`C.CBytes`). Rust copies and frees them with `libc::free`. Rust's
     test roundtrip exports do the mirror image (`libc::malloc` → `C.free`).
     Inputs are borrowed for the duration of the call only.
   - Panic containment: every `//export` Go shim MUST recover panics and
     return `LANCE_GO_CALL_ERROR` with the message as payload. Every Rust C
     export MUST carry the `ffi_guard` attribute; the crate uses unwinding
     only so that guard can convert Rust panics to `ErrInternal`. No panic
     may cross the language boundary.
   - Release semantics: `GoPlugin` is a borrowed, `Copy` handle with no
     `Drop`; synchronous callbacks remain Go-owned. Native caches use the
     `Arc`-backed `OwnedGoPlugin`, whose final drop releases the registration
     after all native clones disappear. `releasePlugin` /
     `go_handle_release` are idempotent. Stale handles fail with a clean
     "no plugin registered" error (locked-map registry, NOT `cgo.Handle`,
     whose `Value()` panics on invalid handles).
   - Any change to `LanceGoVTable`'s layout or conventions must bump
     `LANCE_GO_VTABLE_ABI_VERSION` on both sides (Rust const + the C
     helper in `callbacks_init.go` reads it from the header).

## Platform notes

- macOS link flags: `-framework Security -framework CoreFoundation
  -framework SystemConfiguration -framework IOKit`. Linux: `-lm -ldl
  -lpthread`. Both are baked into `lance/lance.go` `#cgo` directives;
  consumers only need to provide the native library search path.
- The macOS linker may warn about `LC_DYSYMTAB` on the static lib (benign).

## Commit conventions

- Imperative subject, <= 72 chars, no trailing period. Body explains *why*
  when the diff doesn't.
- Rust changes that alter the FFI surface must include the regenerated
  `include/lance_go.h` in the same commit.
- Never commit `rust/target/`, `dist/`, or downloaded artifacts
  (`lib/{os}_{arch}/`).
- Run `make lint` and `make test` before committing, and `make fmt` to fix
  formatting.
- Keep agent-authored work squashed into a single commit unless the user
  explicitly requests otherwise.
- Do not add agents as commit co-authors.
